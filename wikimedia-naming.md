# On Wikimedia naming choices (or: a missed opportunity, thrice over)

There's a page on MediaWiki.org cheekily titled
["Differences between Wikipedia, Wikimedia, MediaWiki, and wiki"](https://www.mediawiki.org/wiki/Differences_between_Wikipedia,_Wikimedia,_MediaWiki,_and_wiki) —
colloquially known as "wikipmediawiki" — whose very existence is a testament to how
confusing the naming of the Wikimedia movement's core projects has become. I've been
thinking about this for a while, and I think it's worth spelling out where things went
wrong, and what could have been done differently.

## Wikimedia

When the Wikimedia Foundation was created in June 2003, the projects it was meant to
oversee were Wikipedia, Wiktionary, and shortly after — in this order — Wikiquote,
Wikibooks, and Wikisource. These are all textual content projects of various kinds.
Unless Jimmy Wales meant "media" in the sense of a vehicle for conveying information
rather than the content itself (i.e. multimedia), the name "Wikimedia" was a somewhat
odd fit. Either way, it's way too similar to "Wikipedia", and has been a source of
perennial confusion ever since.

Something like **Wikiculture Foundation** would have been more appropriate, capturing
the breadth of human knowledge and expression the movement aims to cover. Other options
along those lines could have been **OpenWiki Foundation** or **FreeCulture Foundation**
— though the latter would have clashed with Lawrence Lessig's similarly-named project,
which launched around the same time.

## MediaWiki

This one is particularly unfortunate: "MediaWiki" is quite literally an inversion of
"Wikimedia", which compounds the confusion rather than alleviating it. The name was
suggested in a single mailing list post by Wikipedia contributor Daniel Mayer in July
2003 — as a play on "Wikimedia" — and it just... stuck, with no real deliberation. The
result is a name that is neither descriptive nor distinctive.

A more fitting alternative would have been **PhaseWiki**, following the naming scheme
of MediaWiki's early incarnations ("phase II", "phase III") — a name with some lore to
it, rewarding curiosity without demanding explanation. Some other options worth considering:

- **WikiMod** — pays homage to the UseModWiki origins while remaining plausibly
  interpretable as "wiki module" by those unfamiliar with the history
- **LibreWiki** — links to both the free/open-source nature of the software and its
  purpose of serving open content projects
- **WikiCore** — self-explanatory and pleasingly concise (though it turns out to be a
  name with trademark issues elsewhere, which is exactly what blocked it in the Abstract
  Wikipedia naming contest)
- **PageForge** — drops the "wiki" prefix, but evokes directly the process the software
  enables; perhaps too big a departure

## Wikimedia Commons

This one is, as far as I know, the only major Wikimedia content project that doesn't
use a single compound word for its name, and the only one that doesn't start with
"Wiki". **Wikicommons** is a common (heh) informal alternative, but something more
connected to the actual content it hosts — multimedia files — would be nicer.

Of course, if "Wikimedia" had been freed from its association with the foundation, it
would be a perfect fit here. A Wikimedia repository of media files: the name would
finally make sense. But given the historical baggage, such a three-way swap would be
wholly impracticable at this point — that ship has long sailed 🥲.

As a more realistic alternative, **Wikifiles** would work surprisingly well: it's
intuitive, and "File" is already the namespace used for the content it hosts. Other
options worth a mention: **MediaCommons** would be both clearer and less cumbersome than
the current name, while changing almost nothing (except, admittedly, the "Wiki" prefix,
which matters more than it might seem). On the more evocative end of the spectrum,
**WikiShow** is an interesting contender — it connects to the actual *purpose* of the
media rather than just the mechanism of storing central copies of files for reuse across
sites. The slight awkwardness is that it's a verb, while all the other project names are
nouns (encyclopedia, dictionary, quotes, books, species, voyages...).

---

None of this is particularly actionable, of course. These names are deeply entrenched,
and the cost of changing any of them would far outweigh any clarity gained. But it's
fun to imagine an alternate universe where someone paused, in the summer of 2003, and
thought a little harder before hitting send.
